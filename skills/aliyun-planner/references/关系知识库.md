### 关系类型标准化说明
所有关系统一使用以下5种类型：

1. **contains** (包含)
   - 表示一个资源包含另一个资源
   - 例如：VPC包含VSwitch、服务器组包含后端服务器

2. **depends_on** (依赖)
   - 表示一个资源依赖于另一个资源
   - 例如：ECS依赖于VPC、SLB依赖于VPC

3. **attached_to** (附着/绑定)
   - 表示一个资源物理或逻辑附着到另一个资源
   - 例如：ECS挂载磁盘、EIP绑定到ECS

4. **connected_to** (连接)
   - 表示两个资源之间存在连接关系
   - 例如：SLB连接到后端ECS、应用访问RDS

5. **configured_with** (配置)
   - 表示一个资源配置了某种功能或策略
   - 例如：监听器配置ACL、规则配置转发动作、负载均衡配置WAF

## 1. ECS - 弹性计算服务
核心资源： 实例 (Instance)
角色： 计算单元，承载应用程序。

### 核心关系
网络归属 (位于)
关系： depends_on (ECS -> VPC)
路径： ECS -> VSwitch -> VPC
描述： ECS实例必须部署在VPC内的VSwitch中，继承VPC的网络隔离。
关联字段： VSwitchId (ECS) -> VpcId (VPC)。
查询示例： "ECS i-123所在的VPC"、"VPC vpc-xxx下有哪些ECS"。

存储挂载 (挂载)
关系： attached_to (ECS -> Disk)
描述： ECS可以挂载多块云盘（系统盘/数据盘），磁盘必须挂载到ECS方可使用。
关联字段： InstanceId (Disk)。
查询示例： "ECS i-123挂载了哪些磁盘"、"磁盘 d-123挂载在哪台ECS上"。

安全配置 (配置)
关系： configured_with (ECS -> SecurityGroup)
描述： ECS实例必须关联至少一个安全组，用于控制网络访问规则。
关联字段： SecurityGroupIds (ECS)。
查询示例： "ECS i-123关联了哪些安全组"。

公网访问 (绑定)
关系： attached_to (ECS -> EIP)
描述： ECS可以绑定弹性公网IP（EIP）以实现公网访问。一台ECS可同时绑定多个EIP。
关联字段： AllocationId (ECS中挂载EIP的网卡信息)。
查询示例： "ECS i-123绑定了哪些EIP"、"EIP eip-xxx绑定了哪个资源"。

### 扩展关系
监控 (被监控)
关系： configured_with (ECS -> CMS)
数据： CPU、内存、磁盘IO、网络流量等指标。

存储快照 (来源于)
关系： depends_on (ECS SystemDisk -> Snapshot)
镜像 (基于)
关系： depends_on (ECS -> Image)

负载均衡后端 (作为)
关系： connected_to (SLB/ALB -> ECS)

函数计算挂载 (使用)
关系： attached_to (FC -> NAS attached to ECS) 间接关系

容器节点 (作为)
关系： contains (ACK Cluster -> ECS)

## 2. RDS - 关系型数据库
核心资源： 数据库实例 (DBInstance)
角色： 托管式关系型数据库服务（MySQL, PostgreSQL, SQL Server等）。

### 核心关系
网络归属 (位于)
关系： depends_on (RDS -> VPC)
路径： RDS -> VSwitch -> VPC
描述： RDS实例必须部署在VPC内，保证网络隔离和安全。
关联字段： VSwitchId (RDS) -> VpcId (VPC)。
查询示例： "RDS rm-xxx所在的VPC"。

连接与访问
关系： connected_to (ECS/FC/ACK -> RDS)
描述： 应用程序（运行于ECS、FC或ACK容器中）通常内网连接RDS实例。
关联： 基于共同VPC或VPC对等连接，通过RDS的ConnectionString或Endpoint。
查询示例： "哪些ECS连接到了RDS rm-xxx"。

备份存储 (存储于)
关系： configured_with (RDS -> OSS)
描述： RDS的物理备份和日志备份可存储在OSS中。
查询示例： "RDS rm-xxx的备份存在哪个OSS Bucket"。

高可用/只读实例 (关联)
关系： contains (Primary RDS -> Secondary/ReadOnly RDS)
描述： 主实例与灾备实例、只读实例之间的复制关系。
查询示例： "RDS rm-xxx的只读实例有哪些"。

### 扩展关系
监控 (被监控)
关系： configured_with (RDS -> CMS)
数据： 连接数、QPS、TPS、磁盘空间、IOPS等。

白名单 (控制访问)
关系： configured_with (RDS -> IP白名单或安全组)

### 核心关系

关系： depends_on (PolarDB -> VPC)
描述： 计算节点（DBNode）部署在VPC中。

存储层 (使用)

描述： 核心特性，存储层独立且共享，实现快速扩展。

读写节点与只读节点 (关联)

## 6. SLB - 传统负载均衡
核心资源： 负载均衡实例 (LoadBalancer)
角色： 四层（TCP/UDP）和七层（HTTP/HTTPS）流量分发。

### 核心关系
网络部署 (位于/面向)
关系： depends_on (SLB -> VPC)
描述： SLB可以是私网（部署在VPC内）或公网类型。
关联字段： VpcId (私网SLB)。

后端服务 (分发至)
关系： connected_to (SLB -> ECS/ENI)
描述： SLB监听器（Listener）将流量转发到后端服务器组（BackendServer）中的ECS实例。
关联字段： BackendServers (包含ECS InstanceId和端口)。
查询示例： "SLB lb-xxx的后端服务器有哪些"。

安全与防护 (受保护)
关系： configured_with (SLB -> WAF/DDoS)
描述： 公网SLB可接入WAF进行Web防护，或由DDoS高防保护。

域名绑定 (关联)
关系： configured_with (SLB Listener -> DNS Record)
描述： 七层监听器通常与DNS记录绑定，通过域名访问。

### 转发规则与访问控制 (七层HTTP/HTTPS)
监听器规则 (配置)
关系： configured_with (SLB Listener -> ForwardingRules)
描述： 七层监听器可以配置多个转发规则，基于域名、路径等信息将流量分发到不同的后端服务器组。
关联字段： RuleList (包含规则名称、域名、路径、后端服务器组ID)。
查询示例： "SLB监听器 lsn-xxx 有哪些转发规则"、"规则 rule-xxx 转发到哪个后端服务器组"。

访问控制 (应用)
关系： configured_with (SLB Listener -> ACL)
描述： 监听器可以关联访问控制列表（ACL），用于黑白名单IP限制。
关联字段： AclId (监听器配置)。
查询示例： "监听器 lsn-xxx 的ACL包含哪些IP"、"ACL acl-xxx 被哪些监听器使用"。

访问控制类型 (设置)
关系： configured_with (SLB Listener -> ACL Type)
描述： ACL可以是白名单或黑名单模式。
- 白名单：只允许ACL中的IP访问
- 黑名单：拒绝ACL中的IP访问

规则与ACL的关系 (协同)
关系： configured_with (SLB ForwardingRules -> ACL)
描述： 转发规则和ACL可以同时作用于监听器，流量首先通过ACL过滤，再匹配转发规则。
优先级： ACL过滤（第一层） -> 转发规则匹配（第二层） -> 后端服务器选择

### 转发规则与访问控制 (四层TCP/UDP)
监听器访问控制 (应用)
关系： configured_with (SLB TCP/UDP Listener -> ACL)
描述： 四层监听器同样支持访问控制列表，通过IP黑白名单限制访问。
关联字段： AclId (四层监听器配置)。

健康检查 (监控)
关系： configured_with (SLB Listener -> BackendServer Health)
描述： 监听器通过健康检查探测后端服务器的可用性，自动摘除不健康的服务器。

## 7. ALB - 应用型负载均衡
核心资源： 负载均衡实例 (LoadBalancer)
角色： 七层高级负载均衡，支持基于内容的路由。

### 核心关系
网络部署 (位于)
关系： depends_on (ALB -> VPC)
描述： ALB通常部署在VPC内，通过EIP或共享带宽提供公网服务。

后端服务 (分发至)
关系： connected_to (ALB -> ECS/ECI/FC/IP)
描述： 支持更丰富的后端类型，包括ECS、ECI容器实例、FC函数、IP等。
查询示例： "ALB alb-xxx的后端服务器组包含哪些资源"。

安全与防护 (受保护)
关系： configured_with (ALB -> WAF)

域名与路由规则 (关联)
关系： configured_with (ALB -> DNS Host + Path Rules)

### 转发规则体系 (核心特性)
监听器 (入口)
关系： configured_with (ALB -> Listeners)
描述： ALB实例可以配置多个监听器，每个监听器监听特定的协议和端口。
关联字段： ListenerPort, ListenerProtocol (HTTP/HTTPS/QUIC)。
查询示例： "ALB alb-xxx 有哪些监听器"、"监听器 lsn-xxx 监听哪个端口"。

转发规则 (路由)
关系： configured_with (ALB Listener -> Rules)
描述： 监听器可以配置多个转发规则，每个规则定义匹配条件和转发动作。
关联字段： RuleConditions (匹配条件) + RuleActions (转发动作)。
查询示例： "监听器 lsn-xxx 有多少条转发规则"、"规则 rule-xxx 匹配哪些条件"。

规则条件 (匹配)
关系： configured_with (ALB Rule -> Conditions)
描述： 转发规则支持多种匹配条件：
- 域名 (Host): 基于请求头中的Host字段
- 路径 (Path): 基于URL路径
- HTTP头部 (Header): 基于请求头中的键值对
- 查询字符串 (QueryString): 基于URL查询参数
- HTTP方法 (Method): GET/POST/PUT/DELETE等
- Cookie: 基于请求中的Cookie
- 源IP (SourceIp): 基于客户端IP地址
查询示例： "哪些规则匹配域名 example.com"、"哪些规则匹配路径 /api/*"。

规则动作 (转发)
关系： configured_with (ALB Rule -> Actions)
描述： 匹配成功后执行的动作：
- 转发至服务器组 (ForwardGroup): 转发到后端服务器组
- 重定向 (Redirect): 30X重定向到指定URL
- 固定响应 (FixedResponse): 返回固定内容
- 请求头重写 (Rewrite): 修改请求头后转发
查询示例： "规则 rule-xxx 转发到哪个服务器组"、"规则 rule-xxx 是否重定向"。

### 服务器组与后端资源
服务器组 (汇聚)
关系： contains (ALB ServerGroup -> BackendServers)
描述： 服务器组包含多个后端服务器，支持多种后端类型：
- ECS实例: 云服务器
- ECI实例: 容器实例
- IP地址: VPC内的IP地址
- 函数计算: Serverless函数
关联字段： ServerGroupId, ServerType, BackendList。
查询示例： "服务器组 srg-xxx 包含哪些后端"、"哪些规则转发到服务器组 srg-xxx"。

服务器组类型 (分类)
关系： contains (ALB ServerGroup -> ServerTypes)
描述： 服务器组分为不同类型：
- 实例类型: 添加ECS或ECI实例
- IP类型: 添加VPC内的IP地址
- IP组类型: 可以跨VPC添加后端

### 访问控制列表 (ACL)
访问控制策略 (应用)
关系： configured_with (ALB Listener -> AclEntries)
描述： 监听器可以配置访问控制策略，限制客户端IP访问。
关联字段： AclEntries (包含IP/CIDR条目)。

ACL条目 (条目)
关系： contains (ALB ACL -> Entry)
描述： 每个ACL条目包含：
- IP地址或CIDR网段
- 条目描述
查询示例： "监听器 lsn-xxx 的ACL包含哪些IP"、"IP 1.2.3.4 是否在ACL中"。

ACL与规则的协同 (多层控制)
关系： configured_with (ALB Listener -> ACL -> Rules -> ServerGroup)
描述： 访问控制和转发规则形成多层流量处理：
1. 第一层：ACL过滤 (基于源IP)
2. 第二层：规则匹配 (基于域名、路径、HTTP头等)
3. 第三层：转发到服务器组
优先级： ACL优先级最高，规则按优先级顺序匹配。

### 高级路由特性
规则优先级 (排序)
关系： configured_with (ALB Rules -> Priority)
描述： 转发规则按优先级顺序匹配，优先级为1-10000，数值越小优先级越高。
查询示例： "监听器 lsn-xxx 的规则优先级顺序"。

规则转发动作 (分发策略)
关系： configured_with (ALB Rule -> ForwardAction)
描述： 转发到服务器组时支持多种分发策略：
- 轮询 (RoundRobin)
- 最小连接数 (LeastConnection)
- 源IP哈希 (SourceIpHash)
- 一致性哈希 (consistent_hash)
查询示例： "规则 rule-xxx 使用什么负载均衡算法"。

会话保持 (状态)
关系： configured_with (ALB Rule -> ClientSession)
描述： 转发规则可以配置会话保持，确保同一客户端的请求转发到同一后端。
关联字段： SessionEnabled, SessionTimeout。
查询示例： "规则 rule-xxx 是否启用了会话保持"。

## 8. VPC - 专有网络
核心资源： 专有网络 (VPC)
角色： 云上私有网络环境的核心枢纽，提供隔离、连接。

### 核心关系 (包含)
包含子网
关系： contains (VPC -> VSwitch)
描述： VPC由至少一个VSwitch（交换机）组成，每个VSwitch属于一个可用区。
查询示例： “VPC vpc-xxx下有哪些VSwitch”。

包含路由表
关系： contains (VPC -> RouteTable)
描述： 控制VPC内流量的转发规则。

包含安全组
关系： contains (VPC -> SecurityGroup)
描述： 安全组是VPC级别的虚拟防火墙。

包含网络ACL
关系： contains (VPC -> NetworkACL) 可选
描述： 子网级别的无状态访问控制列表。

### 核心关系 (承载)
承载计算资源
关系： contains (VPC -> ECS/RDS/Redis/MongoDB/PolarDB/SLB/ALB/ACK Node...)
描述： 绝大多数云产品实例都部署在VPC内，通过VSwitch连接。
查询示例： "VPC vpc-xxx内有哪些云资源"。

连接公网
关系： connected_to (VPC -> NAT Gateway / EIP / Internet Gateway)

连接其他网络
关系： connected_to (VPC -> Other VPC) - VPC对等连接
关系： connected_to (VPC -> VPN Gateway / Express Connect) - 混合云

## 9. OSS - 对象存储
核心资源： 存储空间 (Bucket)
角色： 海量、安全、低成本的对象存储服务。

### 核心关系
作为数据源/目的地
关系： connected_to (OSS -> ECS/FC/ACK/DataWorks/...)
描述： OSS被广泛用于存储静态文件、备份、日志、大数据源等。
查询示例： "哪些ECS应用向Bucket my-bucket写入数据"。(通常需结合日志或监控推断)

存储备份
关系： configured_with (OSS -> RDS/Redis/ECS Disk Snapshot)
描述： 多个服务的备份功能使用OSS作为存储后端。

托管静态网站
关系： configured_with (OSS Bucket -> DNS Record)
描述： 结合CDN和DNS，OSS可以托管静态网站。

日志存储
关系： configured_with (OSS -> SLS Logstore 投递目标 / ECS日志直传)

## 10. NAS - 文件存储

角色： 共享文件存储，支持标准文件协议。

### 核心关系
被计算资源挂载
关系： attached_to (NAS -> ECS/FC/ACK/Container)
描述： NAS文件系统通过挂载点（位于VSwitch）被一个或多个ECS实例或容器挂载，实现文件共享。
关联字段： MountTarget 的 VSwitchId 和访问域名/IP。
查询示例： "NAS文件系统 fs-xxx 被哪些ECS挂载"。

网络归属
关系： depends_on (NAS MountTarget -> VSwitch -> VPC)

## 11. EIP - 弹性公网IP
核心资源： 弹性公网IP地址 (EIP)
角色： 独立的公网IP资源，可动态绑定和解绑。

### 核心关系
绑定至资源
关系： attached_to (EIP -> ECS/NAT Gateway/SLB/ALB)
描述： EIP可以绑定到拥有公网网卡的云资源上，为其提供固定的公网访问入口或出口。
关联字段： InstanceId (EIP关联信息)。

共享带宽 (关联)

## 12. FC - 函数计算
核心资源： 服务 (Service)、函数 (Function)
角色： 事件驱动的全托管Serverless计算服务。

### 核心关系
网络配置 (位于)
关系： depends_on (FC Service -> VPC) 可选
描述： 函数可以配置VPC访问，从而访问VPC内的RDS、Redis等资源。
关联字段： VpcConfig (FC Service)。

访问其他服务
关系： connected_to (FC Function -> RDS/Redis/OSS/MNS/...)
描述： 函数逻辑中常调用其他云服务API或访问其资源。

事件源 (由...触发)
关系： configured_with (FC Function -> OSS Event/Log Trigger/Timer/API Gateway/...)
查询示例： "函数 my-func 由哪些事件源触发"。

存储挂载 (使用)
关系： attached_to (FC -> NAS) 通过VPC配置挂载NAS

## 13. ACK - 容器服务
核心资源： 集群 (Cluster)、节点池 (NodePool)
角色： 托管式Kubernetes容器服务。

### 核心关系
基础设施依赖
关系： depends_on (ACK Cluster -> VPC & VSwitch & ECS/SecurityGroup)
描述： ACK集群的Master和Worker节点（ECS）部署在用户VPC中。
查询示例： "ACK集群 ck-xxx 使用了哪些VSwitch和ECS节点"。

负载均衡
关系： connected_to (ACK Service/Ingress -> SLB/ALB)
描述： Kubernetes Service（类型为LoadBalancer）或Ingress会自动创建或关联SLB/ALB实例。

存储
关系： attached_to (ACK Pod -> Cloud Disk / NAS / OSS Flexvolume)
描述： Pod可以挂载云盘、NAS或OSS作为持久化存储。

配置与密码管理
关系： configured_with (ACK -> KMS/Secrets Manager) 间接

## 附录：ALB与SLB转发规则对比

### 转发规则架构对比

**SLB (传统负载均衡)**
- 七层监听器支持域名和路径转发
- 转发规则相对简单，主要基于域名和路径
- 一个监听器可以配置多个转发规则
- 规则匹配后转发到不同的后端服务器组
- ACL与规则协同：ACL过滤 -> 规则匹配 -> 后端选择

**ALB (应用型负载均衡)**
- 更强大的转发规则引擎
- 支持丰富的匹配条件：域名、路径、HTTP头、查询字符串、Cookie、源IP、HTTP方法
- 支持多种转发动作：转发、重定向、固定响应、请求头重写
- 规则优先级可配置（1-10000）
- 服务器组支持多种后端类型：ECS、ECI、IP、FC
- ACL与规则协同：ACL过滤 -> 规则匹配（按优先级） -> 服务器组选择
